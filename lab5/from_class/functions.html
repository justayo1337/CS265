The Exercises

  1. Write a function, arrAvg, which takes an array of doubles , along with the array's length (as a size_t), and returns the arithmetic mean, as a double. Place this in a C file called func1.c. Include a main function that reads floats fromstdin, one per line, and loads an array. Assume no more than 100 inputs. Test your function.

  2. The easiest way to visit each digit in an integer is to visit them from least- to most- significant (right-to-left), using modulus and division.

    E.g., (working in decimal) 327 % 10 is 7. We record 7, then divide by 10, so we can now process 32.

    To convert an integer on [0, 9] to a character, just add the character '0': 

    char c ;
int i = 9 ;   // for example
c = (char) i + '0' ;

    Write a function called void int2string( int n, char answer[] ), which parses up non-negative n (base 10), writes the string into array pointed to by answer. Remember to terminate the string properly. You may (you really have to) assume that answer is an array large enough to hold the result.

    Your function may not call any functions to help you.

    Since you'll be getting the digits in reverse order, you'll want a temporary local array. You may assume a 4-byte integer, which, base 10, will have, at most, 10 digits. Make sure your function handles an input of 0.

    Place your function into a file called func2.c. Include a main function which reads integers from stdin, one per line (until EOF is signalled).
