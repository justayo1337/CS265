Lab – C Strings

The directory for this lab is ~kschmidt/public_html/CS265/Labs/C .

Introduction

We will test your code using the gcc compiler.

The Exercises

  1. Write a function, capitalise, which takes a null-terminated string (char*), and replaces each lower-case letter with its upper-case counterpart.

    In ctype.h find functions which might help you: 

    int isalpha( char ) ;
int islower( char ) ;
int toupper( char ) ;

    You may not use square brackets, [], anyhwere in your function (other than in comments).

  2. Write a program, circle.c, which expects two command-line arguments, a name (as a string), and a radius (as a double). You will then print the circle's name and area, as demonstrated below.

    If the number of command-line arguments is not right, print an appropriate message to stderr, and return a non-zero status. You may assume that if 2 arguments are supplied, the 2nd argument will be safely converted to a double.

    In stdlib.h find strtod which will parse the double for you. Use the man pages to see its use. This function replaces the older atof function.

    Do not approximate Pi yourself. Use the language to calculate it, or use M_PI, defined in math.h .

    Print the area using scientific notation, with 4 significant digits.

    Here are some example runs: 

    $ ./circle "The Hulk"
Need 2 arguments, a name and a radius.

$ ./circle Samwise 17.3
Samwise, your area is 9.402e+02 units square
