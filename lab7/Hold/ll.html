Linked lists

For all of these, draw pictures, see what's happening. Start by solving a general case, test it. Then consider any border cases which you may need to handle, tackle them imcrementally.

This one is a little tricky. You have some border conditions to test for:

  • The first node is to be removed
  • The last node is to be removed
  • An node to be removed is followed by a node to be removed

Again, write this in stages. For example, maybe handle these cases in turn:

  1. Just a single node to be removed, buried in the middle
  2. Just a single node to be removed, at the front
  3. Just a single node to be removed, at the end
  4. Just a single node to be removed, the only one
  5. More than one target, not adjacent
  6. Several targets, some adjacent
